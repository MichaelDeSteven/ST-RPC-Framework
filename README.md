# ST-RPC-Framework
基于nacos实现的一款简单的、轻量、拥有服务治理能力的rpc框架
## 特性

* 实现了基于 Java 原生 Socket 传输与 Netty 传输两种网络传输方式
* 实现了可配置的动态参数线程池
* 实现了四种序列化算法，Protobuf、Hessian、Kryo、Json
* 实现了多种负载均衡算法，随机、轮转、一致性哈希等
* 服务端提供了自动注册服务机制
* 底层网络传输为TCP，实现了自定义的通信协议
* 支持一个接口多个实现
* 增加Netty心跳机制
